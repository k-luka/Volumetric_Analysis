# Presentation Next Steps

## Goal

Show the doctor a small, credible alpha: one or more scans go in, segmentation/QC outputs and an Excel volume report come out.

## Before The Meeting

1. Rerun the tutorial scan with the current config.
2. If allowed, run 1-3 representative de-identified study scans.
3. Bring the Excel report plus both QC PNGs.
4. Bring `docs/CURRENT_SPEC.md` as the meeting notes.
5. Prepare a short methods paragraph that cites FastSurfer/FastSurferVINN.

## Demo Packet

- One input filename.
- One Excel output row.
- One multi-color QC image.
- One binary QC image.
- Current assumptions and questions for the doctor.
- Current limitations: alpha workflow, NIfTI-only input, one example QC output, no UI yet.

## Do Not Claim Yet

- Do not claim study readiness until the volume definition and QC expectations are confirmed.
- Do not present the segmentation model as original work.
- Do not ask doctors to install or run the command-line workflow directly.
