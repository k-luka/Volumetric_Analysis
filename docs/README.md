# Documentation Index

This folder is the source of truth for project state, plans, and research notes.
The root `README.md` stays as the short user-facing entry point.

## Start Here

- `STATE.md` - current app state, completed work, environment notes, and verification commands.
- `NEXT_STEPS.md` - ordered MVP work queue from the current point forward.
- `ARCHITECTURE.md` - agent-oriented system map and file/API table of contents.

## Product And Research Context

- `CURRENT_SPEC.md` - current alpha workflow, method, outputs, scientific assumptions, and open study/license questions.
- `PRODUCT_VISION.md` - broader product direction and future deployment thinking.
- `DEMO_GUIDE.md` - demo prep, click path, and talking points for a doctor/research stakeholder.

## Engineering Planning

- `TECHNICAL_DEBT.md` - engineering debt and lower-priority technical improvements.
- `OOD_BATCH_APP_PLAN.md` - HiPerGator Open OnDemand batch app plan.
- `SUBMIT_WIZARD_PLAN.md` - local submit/wizard workflow plan.
- `AGENT.md` - concise context for future coding-agent sessions.
